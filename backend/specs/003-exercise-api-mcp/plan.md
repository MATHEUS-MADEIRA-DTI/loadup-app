# Implementation Plan: External Exercise Search API

**Branch**: `003-exercise-api-mcp` | **Date**: 2026-05-12 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/003-exercise-api-mcp/spec.md`

## Summary

This feature adds a `GET /exercises/search` endpoint to the LoadUp API that queries the external API Ninjas Exercises service, maps muscle group names from English to Portuguese, caches results in MongoDB for 7 days, and handles external API failures gracefully. The implementation creates a new NestJS module `src/exercises-api/` that is entirely separate from the existing `src/exercises/` module (spec 001). No existing code is modified except `app.module.ts` (one import added).

## Phase 0: Research

### R-001 — `@nestjs/axios` for external HTTP

**Decision**: Use `@nestjs/axios` (`HttpModule`).
**Rationale**: Already in the NestJS ecosystem; integrates naturally with DI; avoids raw `fetch` boilerplate. Returns Observables converted to Promises via `lastValueFrom`.
**Pattern**: `lastValueFrom(this.httpService.get<ApiNinjasExercise[]>(url, { headers }))` in `exercises-api.client.ts`.

### R-002 — MongoDB manual-TTL cache strategy

**Decision**: Use an `expiresAt: Date` field with a query filter `{ expiresAt: { $gt: new Date() } }` for freshness checks. Expired entries are NOT auto-deleted.
**Rationale**: MongoDB's native TTL index auto-deletes documents, which would destroy the stale-cache fallback required by FR-007. Manual TTL keeps documents indefinitely so the service can return expired results when the external API is unavailable.
**Indexes**: `cacheKey` (unique) for O(1) lookup; `expiresAt` (regular) for freshness queries.

### R-003 — Fail-fast API key validation

**Decision**: Use `registerAs` config factory — throws synchronously during `ConfigModule` initialization if the key is missing.

```typescript
// exercises-api.config.ts
export const exercisesApiConfig = registerAs('exercisesApi', () => {
  const key = process.env.EXERCISES_API_KEY;
  if (!key) throw new Error('[ExercisesApi] EXERCISES_API_KEY is required but not set');
  return { apiKey: key, baseUrl: 'https://api.api-ninjas.com/v1/exercises' };
});
```

NestJS bootstrapping fails with a descriptive message before any request is served.

### R-004 — Cache key normalization

**Decision**: `cacheKey = [name, muscle].filter(Boolean).map(s => s.trim().toLowerCase()).join('|')`
**Examples**: `name=Bench Press` → `"bench press"` | `name=Rosca, muscle=Bíceps` → `"rosca|bíceps"`
**Rationale**: The `muscle` param is received in Portuguese from the frontend. The cache key uses the Portuguese value directly — this keeps the key consistent with what the frontend sends and avoids any dependency on the reverse-mapping step for cache identity.

### R-005 — Input sanitization

**Decision**: `class-validator` decorators on DTO (`@IsOptional() @IsString() @MaxLength(100) @Matches(/^[a-zA-Z0-9\s_-]+$/)`) prevent special characters. Axios URL-encodes query params automatically — no additional sanitization needed.

## Phase 1: Design & Contracts

### Data Model

**Collection**: `exercise_caches`

| Field       | Type                 | Constraints            | Notes                                     |
| ----------- | -------------------- | ---------------------- | ----------------------------------------- |
| `_id`       | ObjectId             | auto                   | Mongoose default                          |
| `cacheKey`  | String               | unique index, required | normalized `name\|muscle`                 |
| `results`   | ExerciseCacheEntry[] | required               | mapped exercise objects (PT muscle names) |
| `createdAt` | Date                 | required               | first cache write timestamp               |
| `expiresAt` | Date                 | index, required        | `createdAt + 7 days`                      |

**ExerciseCacheEntry** (embedded):

| Field          | Type   | Notes                      |
| -------------- | ------ | -------------------------- |
| `name`         | String | exercise name              |
| `muscleGroup`  | String | mapped to Portuguese       |
| `type`         | String | e.g., "strength", "cardio" |
| `equipment`    | String | e.g., "barbell", "none"    |
| `instructions` | String | full instructions text     |

### API Contract

**Endpoint**: `GET /exercises/search`
**Auth**: `Authorization: Bearer <jwt>` (required — uses existing `JwtAuthGuard`)

**Query Parameters**:

| Param    | Type   | Required    | Validation                                                                                                                                         |
| -------- | ------ | ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `name`   | string | conditional | at least one of `name`/`muscle`; max 100 chars; alphanumeric + spaces                                                                              |
| `muscle` | string | conditional | at least one of `name`/`muscle`; max 100 chars; **Portuguese value** (e.g., "Peito", "Bíceps") — mapped to English before forwarding to API Ninjas |

**Success Response 200**:

```json
{
  "results": [
    {
      "name": "Bench Press",
      "muscleGroup": "Peito",
      "type": "Força",
      "equipment": "barbell",
      "instructions": "Lie on a flat bench..."
    }
  ],
  "cached": false,
  "warning": null
}
```

**Cache hit**: `"cached": true, "warning": null`
**Stale cache + API down**: `"cached": true, "warning": "Resultados em cache (API externa indisponível)"`
**No cache + API down**: `"results": [], "cached": false, "warning": "Serviço de exercícios temporariamente indisponível"`

**Error Responses**:

- `400 Bad Request` — neither `name` nor `muscle` provided (class-validator + custom guard)
- `401 Unauthorized` — missing or invalid JWT

### Muscle Group Mapper

Three pure functions — no external dependencies:

```typescript
// src/exercises-api/mappers/muscle-group.mapper.ts
const MUSCLE_MAP: Record<string, string> = {
  chest: 'Peito',
  biceps: 'Bíceps',
  triceps: 'Tríceps',
  back: 'Costas',
  shoulders: 'Ombros',
  abdominals: 'Abdômen',
  quadriceps: 'Perna',
  hamstrings: 'Perna',
  glutes: 'Glúteo',
  calves: 'Perna',
  forearms: 'Bíceps',
  middle_back: 'Costas',
  lower_back: 'Costas',
  lats: 'Costas',
  neck: 'Ombros',
};

// EN -> PT: applied after receiving results from API Ninjas, before caching
export function mapMuscleGroup(value: string): string {
  return MUSCLE_MAP[value.toLowerCase()] ?? value;
}

// PT -> EN: applied before calling API Ninjas, using the Portuguese muscle param from the frontend
const REVERSE_MUSCLE_MAP: Record<string, string> = Object.entries(MUSCLE_MAP).reduce<
  Record<string, string>
>((acc, [en, pt]) => ({ ...acc, [pt.toLowerCase()]: en }), {});

export function mapMuscleGroupToEnglish(ptValue: string): string {
  return REVERSE_MUSCLE_MAP[ptValue.trim().toLowerCase()] ?? ptValue;
}

// EN -> PT: applied to the `type` field after receiving results, before caching
const EXERCISE_TYPE_MAP: Record<string, string> = {
  strength: 'Força',
  cardio: 'Cardio',
  stretching: 'Alongamento',
  powerlifting: 'Powerlifting',
  plyometrics: 'Pliometria',
  olympic_weightlifting: 'Halterofilia',
  strongman: 'Strongman',
};

export function mapExerciseType(value: string): string {
  return EXERCISE_TYPE_MAP[value.toLowerCase()] ?? value;
}
```

> **Note on reverse map collisions**: Multiple English keys map to the same Portuguese value (e.g., `quadriceps`, `hamstrings`, `calves` → `"Perna"`). The reverse map will store only one English key per Portuguese value (last write wins in `reduce`). This is acceptable — API Ninjas `muscle` param accepts any of these values and the user only sees Portuguese. The chosen English key for `"Perna"` will be `"calves"` (last in iteration order). This is a known, documented approximation.

### Service Orchestration Logic

```
search(name?, muscle?):
  1. Build cacheKey from (name, muscle)         // muscle is Portuguese here
  2. Query MongoDB: findOne({ cacheKey, expiresAt > now })
  3. Cache HIT  -> return { results, cached: true, warning: null }
  4. Cache MISS -> try:
       a. muscleEn = muscle ? mapMuscleGroupToEnglish(muscle) : undefined
       b. Call API Ninjas with (name, muscleEn)  // English forwarded to external API
       c. Map each result: muscleGroup = mapMuscleGroup(result.muscle), type = mapExerciseType(result.type)  // EN->PT for both fields
       d. Upsert cache document (cacheKey, mapped results, expiresAt = now+7d)
       e. return { results: mapped, cached: false, warning: null }
     catch ExternalApiError:
       f. Query MongoDB: findOne({ cacheKey })   // stale OK
       g. If stale exists -> return { results, cached: true, warning: "API externa indisponível" }
       h. Else -> return { results: [], cached: false, warning: "Serviço temporariamente indisponível" }
```
