## Summary

This feature adds automatic plateau detection to the LoadUp backend. When a training session is marked as "completed", an event is emitted and a listener asynchronously analyzes the user's `SessionRecord` history for each exercise. A plateau is detected when a user performs the same weight AND same reps (in "working" series) across 4+ consecutive sessions, or over a 2-week window. Results are persisted as `PlateauAlert` documents in MongoDB and served via two read endpoints. Detection never blocks the session completion response.

## Phase 0: Research

### R-001 — `@nestjs/event-emitter` for async event bus

**Decision**: Use `@nestjs/event-emitter` (wraps `EventEmitter2`).
**Rationale**: Native NestJS integration; decorators (`@OnEvent`) are clean; `EventEmitterModule.forRoot()` registers globally; no infrastructure dependency (no Redis/queue). The listener runs in the same process — fast enough for a personal project. The `async: false` default in EventEmitter2 means the emitter does not await the listener; session completion returns immediately.
**Pattern**:

```typescript
// In TrainingSessionService.completeSession():
this.eventEmitter.emit('session.completed', { userId, sessionId, dayOfWeek });

// In SessionCompletedListener:
@OnEvent('session.completed')
async handleSessionCompleted(payload: SessionCompletedEvent): Promise<void> {
  await this.plateauService.runDetectionForUser(payload.userId);
}
```

### R-002 — Detection isolation: failure must not bubble up

**Decision**: The listener wraps the entire detection in `try/catch` and only logs on failure. The event emission in the service is fire-and-forget; NestJS EventEmitter2 does not propagate listener errors back to the emitter by default.

```typescript
@OnEvent('session.completed')
async handleSessionCompleted(payload: SessionCompletedEvent): Promise<void> {
  try {
    await this.plateauService.runDetectionForUser(payload.userId);
  } catch (err) {
    this.logger.error('Plateau detection failed', (err as Error).message);
  }
}
```

### R-003 — Plateau analysis algorithm

**Decision**: Query ALL `TrainingSession` documents for the user, sorted by `date` descending. For each unique `exerciseName` in `SessionRecord` with `seriesType: 'working'`, build a list of `(weight, reps)` tuples ordered by session date descending. Walk the list to count consecutive identical tuples. Plateau triggers at `count >= 4` OR if the date span of the identical window exceeds 14 days.

**Pseudocode**:

```
analyze(userId):
  sessions = TrainingSession.find({ userId }).sort({ date: -1 }).populate records
  exerciseMap = group records by exerciseName (working only)

  for each exerciseName:
    tuples = [(r.weight, r.repsCompleted) for r in records, ordered by date desc]
    consecutiveCount = 1
    while tuples[i] == tuples[0] and i < tuples.length: consecutiveCount++
    dateSpanDays = (sessions[0].date - sessions[consecutiveCount-1].date) / ms_per_day

    isInPlateau = consecutiveCount >= 4 OR (consecutiveCount >= 2 AND dateSpanDays >= 14)
    suggestion = pickSuggestion(tuples, consecutiveCount)
    yield PlateauAnalysisResult { exerciseName, isInPlateau, consecutiveCount, suggestion }
```

### R-004 — Upsert strategy for idempotent detection

**Decision**: Use `findOneAndUpdate({ userId, exerciseName, alertType: 'exercise' }, { $set: { ... } }, { upsert: true })` for each exercise result. Resolved alerts (progression detected) set `active: false, resolvedAt: new Date()`. New/continuing plateaus set `active: true, resolvedAt: null`.

### R-005 — Day-level aggregate alert generation

**Decision**: After all exercise-level upserts, group active exercise alerts by `dayOfWeek`. For each day with `count >= 2`, upsert a day-level alert. For days that no longer qualify, set `active: false`. The day-level `exerciseName` field stores the `dayOfWeek` value for consistency.

### Suggestion Selection Logic

```typescript
// src/plateau/constants/suggestions.ts

export const SUGGESTIONS = {
  INCREASE_WEIGHT: 'Tente aumentar 2.5kg no próximo treino',
  INCREASE_REPS: 'Tente fazer mais 1 repetição na próxima sessão',
  REDUCE_REST: 'Considere reduzir o tempo de descanso entre séries',
  VARY_EXERCISE: 'Tente variar o exercício por 1 semana',
} as const;

export function pickSuggestion(
  tuples: Array<{ weight: number; reps: number }>,
  consecutiveCount: number,
): string {
  if (consecutiveCount >= 5) {
    // alternate between REDUCE_REST and VARY_EXERCISE for severe plateaus
    return consecutiveCount % 2 === 0 ? SUGGESTIONS.REDUCE_REST : SUGGESTIONS.VARY_EXERCISE;
  }
  // Compare with the session before the plateau window
  const current = tuples[0];
  const before = tuples[consecutiveCount] ?? null;
  if (!before) return SUGGESTIONS.INCREASE_WEIGHT; // default when no prior history

  if (current.weight === before.weight) return SUGGESTIONS.INCREASE_WEIGHT;
  return SUGGESTIONS.INCREASE_REPS;
}
```
